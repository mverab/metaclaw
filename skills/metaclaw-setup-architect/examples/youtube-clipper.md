# Example: YouTube Viral Clipper Setup

Complete end-to-end example of a multi-agent setup generated by the Setup Architect.

---

## Setup Brief

**Name**: YouTube Viral Clipper
**Purpose**: Automatically identify, clip, and publish viral short-form content from long-form YouTube videos
**Pipeline**: Monitor channels → Download & transcribe → Identify viral moments → Clip & edit → Publish shorts
**Agents**: 3 — scout, editor, publisher
**Channels**: Telegram (control), Discord (inter-agent)
**Key Tools**: yt-dlp, ffmpeg, whisper, browser, youtube-data-api
**Autonomy**: Semi-auto (auto-clip, human approves before publishing)
**Personality**: Efficient, data-driven, creative eye for viral content

---

## Architecture Blueprint

### Pattern: Pipeline + Human-in-the-Loop

### Agents

| Agent | Role | Model | Channel |
|-------|------|-------|---------|
| scout | Monitor YouTube channels, identify high-potential videos | ollama/llama3.2 | Discord #scouting |
| editor | Download, transcribe, identify clips, cut & format | ollama/qwen2.5-coder:14b | Discord #editing |
| publisher | Review queue, get human approval, publish to platforms | ollama/llama3.2 | Telegram DM |

### Data Flow
```
scout (finds videos) → #scouting channel → editor (clips them) → #editing channel → publisher (queues for approval) → Telegram → human approves → publish
```

### Editor Agent

#### workspace-editor/AGENTS.md
```markdown
# Editor — Operating Instructions

## Role
You download scouted videos, transcribe them, identify the best clip moments, and produce ready-to-publish short-form clips.

## Core Responsibilities
- Monitor Discord #scouting for new scout reports
- Download videos using yt-dlp
- Transcribe using Whisper
- Analyze transcript for viral moments (hooks, punchlines, emotional peaks, surprising statements)
- Cut clips to 30-60 seconds optimized for short-form
- Convert to vertical 9:16 format
- Generate subtitles (large, centered, word-by-word highlight style)
- Pass finished clips to #editing channel for publisher

## Tools Available
- `exec` — run yt-dlp, ffmpeg, whisper commands
- `read` / `write` — manage clip files and transcripts
- `filesystem` — organize clips directory

## Clip Identification Criteria
Rate each potential clip on:
1. **Hook strength** (first 3 seconds grab attention?)
2. **Standalone value** (makes sense without full video context?)
3. **Emotional peak** (surprise, humor, insight, controversy?)
4. **Shareability** (would someone send this to a friend?)
5. **Length** (sweet spot: 30-60 seconds)

## Technical Specs
- Output resolution: 1080x1920 (9:16 vertical)
- Subtitles: White text, black outline, centered bottom third
- Format: MP4 (H.264 + AAC)
- Max file size: 50MB

## Commands Reference

### Download video
```bash
yt-dlp -f "bestvideo[height<=1080]+bestaudio" -o "~/clips/%(id)s.%(ext)s" "{url}"
```

### Transcribe
```bash
whisper "~/clips/{video_id}.mp4" --model medium --output_format srt --output_dir "~/clips/"
```

### Cut clip
```bash
ffmpeg -i "~/clips/{video_id}.mp4" -ss {start} -to {end} -c copy "~/clips/{video_id}_clip_{n}.mp4"
```

### Convert to vertical
```bash
ffmpeg -i "{input}" -vf "crop=ih*9/16:ih,scale=1080:1920" -c:a copy "{output}"
```

### Add subtitles
```bash
ffmpeg -i "{input}" -vf "subtitles={srt}:force_style='FontSize=22,Alignment=2,MarginV=60,BorderStyle=3,Outline=2'" "{output}"
```

## Coordination
Post finished clips to Discord #editing channel:
```
✂️ **CLIP READY**
**Source**: [original video title]
**Clip**: [description of moment]
**Duration**: [length]
**File**: [path to clip file]
**Viral score**: [X/10]
**Suggested caption**: [short caption for posting]
```

## Decision Framework

### Auto-execute
- Downloading and transcribing videos
- Identifying clip moments
- Cutting and formatting clips

### Ask before
- Processing videos longer than 3 hours
- Re-editing previously rejected clips
```

### Custom Skills

#### skills/clip-identifier/SKILL.md
```markdown
---
name: clip_identifier
description: Analyze video transcripts to identify viral-worthy moments for short-form clips. Scores potential clips by hook strength, standalone value, emotional peak, and shareability.
emoji: 🔍
requires:
  bins: [whisper]
---

# Clip Identifier

## When to Use
After transcribing a video, use this to find the best moments to clip.

## Implementation
1. Read the full transcript (SRT format)
2. Scan for these viral indicators:
   - **Strong hooks**: Statements that create curiosity in first 3 seconds
   - **Contrarian takes**: Opinions that challenge conventional wisdom
   - **Emotional peaks**: Moments of humor, surprise, anger, or inspiration
   - **Quotable lines**: Concise statements people would screenshot or share
   - **Story arcs**: Mini-stories with setup + punchline within 60 seconds
3. For each potential clip, define:
   - Start timestamp (2-3 seconds before the hook)
   - End timestamp (cut after the punchline/peak, not during trailing silence)
   - Viral score (1-10)
   - Suggested caption
4. Rank clips by viral score, return top 5

## Output Format
For each clip:
```
**Clip [N]** — Score: [X/10]
- Start: [HH:MM:SS]
- End: [HH:MM:SS]
- Duration: [seconds]s
- Hook: "[first few words that grab attention]"
- Why: [1 sentence on why this moment is viral]
- Caption: "[suggested caption for posting]"
```

## Guardrails
- Minimum clip duration: 15 seconds
- Maximum clip duration: 90 seconds
- Sweet spot: 30-60 seconds
- Never clip mid-sentence — always find natural start/end points
- Avoid clips that require context from the rest of the video
```

## Installation

```bash
# 1. Create agent workspaces
openclaw agents add scout
openclaw agents add editor
openclaw agents add publisher

# 2. Copy configuration
cp openclaw.json ~/.openclaw/openclaw.json

# 3. Copy workspace files (for each agent)
cp -r workspace-scout/* ~/.openclaw/workspace-scout/
cp -r workspace-editor/* ~/.openclaw/workspace-editor/
cp -r workspace-publisher/* ~/.openclaw/workspace-publisher/

# 4. Install dependencies
pip install yt-dlp openai-whisper
brew install ffmpeg

# 5. Set up Ollama (LLM — no API key needed)
# Install: https://ollama.com
ollama serve &
ollama pull llama3.2
ollama pull qwen2.5-coder:14b
export OLLAMA_BASE_URL="http://localhost:11434"

# SearXNG is included in docker-compose — no setup needed

# 6. Set channel tokens
export TELEGRAM_BOT_TOKEN="your-telegram-bot-token"
export DISCORD_BOT_TOKEN="your-discord-bot-token"

# Optional: use z.ai cloud models instead of local Ollama
# export ZAI_API_KEY="your-z.ai-api-key"
# export ZAI_BASE_URL="https://api.z.ai/v1"

# 7. Create Discord server with channels: #scouting, #editing
# 8. Create Telegram bot via BotFather

# 9. Configure channels
openclaw channels login --channel telegram --account clipper-bot
openclaw channels login --channel discord --account clipper-server

# 10. Create clips directory
mkdir -p ~/.openclaw/workspace-editor/clips

# 11. Restart and verify
openclaw gateway restart
openclaw agents list --bindings
openclaw channels status --probe
```

### Required
- **Ollama** — install at ollama.com, pull `llama3.2` and `qwen2.5-coder:14b`
- **Telegram Bot Token** (via @BotFather)
- **Discord Bot Token** (via Discord Developer Portal — enable Message Content Intent)
- **YouTube channels to monitor** — add URLs to scout's memory/SYSTEM.md

### Optional cloud upgrades
- **z.ai** — set `ZAI_API_KEY` + `ZAI_BASE_URL` and change model to `zai/glm-4`
- **Anthropic / OpenAI** — set the corresponding API key and change model

SearXNG is included in docker-compose. Zero API keys required for the base setup.
